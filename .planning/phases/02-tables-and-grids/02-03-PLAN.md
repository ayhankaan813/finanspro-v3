IMPORTANT BACKGROUND: The dashboard layout container at apps/frontend/src/app/(dashboard)/layout.tsx line 79 has:
```tsx
<div className="container px-3 py-1 sm:py-6 lg:px-8 lg:py-8">
```
The `px-3` (12px) on mobile is the padding that `-mx-3` offsets to make Cards edge-to-edge.

**Pattern for TABL-02 (edge-to-edge Card on mobile):**
Wrap the Card in a div with `-mx-3 sm:mx-0`. This pulls the Card flush to viewport edges on mobile while restoring normal position at sm: breakpoint and above. The Card itself keeps all its existing classes — no changes to Card className needed.

```tsx
{/* Before */}
<Card className="rounded-3xl border-0 shadow-xl overflow-hidden">
  ...
</Card>

{/* After */}
<div className="-mx-3 sm:mx-0">
  <Card className="rounded-3xl border-0 shadow-xl overflow-hidden">
    ...
  </Card>
</div>
```

WHY a wrapper div instead of modifying Card className: The Card already has many classes including overflow-hidden for rounded corners. Adding -mx-3 to the Card itself works too, but a wrapper div is cleaner — it separates positioning concern (wrapper) from visual concern (Card). Either approach is acceptable; the wrapper div is preferred for clarity.

**Pattern for TABL-03 (whitespace-nowrap on cells):**
Add `whitespace-nowrap` to `<th>` and `<td>` elements that contain dates, financial numbers, short labels, and badges. Do NOT add it to description/notes cells (use `truncate` on those instead). Since most cells in financial tables contain numbers and short text, add it broadly.

```tsx
{/* Before */}
<th className="py-3 px-4 text-sm font-semibold text-right">Yatırım</th>
<td className="py-3 px-4 text-right text-sm font-bold">...</td>

{/* After */}
<th className="py-3 px-4 text-sm font-semibold text-right whitespace-nowrap">Yatırım</th>
<td className="py-3 px-4 text-right text-sm font-bold whitespace-nowrap">...</td>
```

NOTE: Elements with the `.font-amount` class already have `white-space: nowrap` from globals.css (Phase 1 GLOB-05). No need to add `whitespace-nowrap` redundantly to those elements. Focus on `<th>` and `<td>` elements that do NOT already have `font-amount`.

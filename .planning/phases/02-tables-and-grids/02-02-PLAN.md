Fix the card grid breakpoints on all four entity list pages (sites, partners, financiers, external-parties) so that the two-column layout activates at 640px (sm:) instead of 768px (md:). This fills the wasted space between 640-767px where cards currently display in a single oversized column.

Purpose: All four list pages use `md:grid-cols-2` as their first multi-column breakpoint. Between 640px and 767px, each card stretches the full width — too wide for a single card, not wide enough to warrant it. Switching to `sm:grid-cols-2` gives a natural two-column layout at tablet-portrait widths. The dashboard page and sites header stats grid already have appropriate responsive behavior and need no changes.

Output: Four files each have one className string change — `md:grid-cols-2` becomes `sm:grid-cols-2`. Higher breakpoints (lg:, xl:) remain untouched.

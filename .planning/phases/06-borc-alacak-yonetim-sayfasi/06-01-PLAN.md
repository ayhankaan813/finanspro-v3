Refactor the backend Debt system from financier-only to general-purpose polymorphic entity tracking.

Purpose: The UAT revealed the user needs a general-purpose tracking account system where debts can exist between ANY entity types (Partner ↔ ExternalParty, Organization ↔ Partner, Financier ↔ Partner, etc.), not just between financiers. This is the foundational backend change that all frontend work depends on.

Output: Updated Prisma schema with DebtEntityType enum, refactored debt service with polymorphic entity resolution, updated Zod schemas, and updated API endpoints.
</objective>

  **A. debt.schema.ts — Add entity type validation:**

  1. Define a Zod enum for debt entity types:
     ```typescript
     const debtEntityTypeEnum = z.enum(['FINANCIER', 'PARTNER', 'EXTERNAL_PARTY', 'ORGANIZATION']);
     ```

  2. Update `createDebtSchema`:
     - Add `lender_type: debtEntityTypeEnum` (required)
     - Add `borrower_type: debtEntityTypeEnum` (required)
     - Keep `lender_id: z.string().uuid()` (remove the old financier-specific message)
     - Keep `borrower_id: z.string().uuid()`
     - Keep `amount` and `description` as-is
     - Add a `.refine()` at the schema level to check: if lender_type === borrower_type, then lender_id !== borrower_id (same type + same entity = invalid)

  3. Update `debtQuerySchema`:
     - Replace `financier_id` with `entity_id: z.string().uuid().optional()`
     - Add `entity_type: debtEntityTypeEnum.optional()`
     - Keep `status`, `page`, `limit`

  **B. debt.service.ts — Polymorphic entity resolution:**

  1. Create a helper function `resolveEntityName(type: DebtEntityType, id: string): Promise<{ name: string; code?: string }>` that:
     - For FINANCIER: `prisma.financier.findUnique({ where: { id }, select: { name: true, code: true } })`
     - For PARTNER: `prisma.partner.findUnique({ where: { id }, select: { name: true, code: true } })`
     - For EXTERNAL_PARTY: `prisma.externalParty.findUnique({ where: { id }, select: { name: true } })` (no code field, set code to undefined)
     - For ORGANIZATION: Return `{ name: 'Organizasyon', code: 'ORG' }` (singleton — no DB lookup needed)
     - Throws NotFoundError if entity not found

  2. Create a helper `validateEntityExists(type: DebtEntityType, id: string): Promise<void>` that:
     - Looks up the entity by type+id
     - Checks `is_active === true` and `deleted_at === null` (for FINANCIER, PARTNER, EXTERNAL_PARTY)
     - For ORGANIZATION: always valid (just verify ORG_ENTITY_ID matches or skip check)
     - Throws NotFoundError/BusinessError if invalid

  3. Rewrite `create()`:
     - Accept the new input shape (with lender_type, borrower_type)
     - Call `validateEntityExists()` for both lender and borrower
     - Store `lender_type`, `borrower_type`, `lender_id`, `borrower_id` in the Debt record
     - After creation, resolve entity names and return enriched response

  4. Rewrite `findAll()`:
     - Update filter: if `entity_id` is given, filter by `OR: [{ lender_id }, { borrower_id }]`
     - If `entity_type` is given, also filter by `OR: [{ lender_type }, { borrower_type }]` (combined with entity_id if both provided)
     - After fetching debts, resolve entity names for all lenders/borrowers via `resolveEntityName()`
     - Return enriched items with `lender: { id, name, code, type }` and `borrower: { id, name, code, type }`

  5. Rewrite `findById()`:
     - After fetching, resolve entity names
     - Return enriched response

  6. Rewrite `pay()` and `cancel()`:
     - These mostly stay the same, but after update, resolve entity names in returned object

  7. Rewrite `getSummary()`:
     - No changes needed — it aggregates across all debts regardless of entity type
     - The existing logic is correct (sums remaining_amount across all ACTIVE debts)

  8. Rewrite `getFinancierSummary()` → rename to `getEntitySummary()`:
     - Fetch ALL active debts (not just financier debts)
     - Fetch all active entities from all 4 tables (financiers, partners, external_parties)
     - For Organization: add a single entry with name 'Organizasyon'
     - Build summary per entity: `{ entity: { id, name, code, type }, total_receivable, total_owed, net_position }`
     - Only include entities that have at least one active debt (as lender or borrower)

  9. Rewrite `getMatrix()`:
     - Fetch all active debts
     - Collect unique entity references (type+id pairs) from all debts
     - Resolve names for all entities
     - Build matrix: `{ entities: [...], matrix: [{ lender: {..., type}, borrower: {..., type}, amount }] }`

  **C. debt.routes.ts — Update route for renamed endpoint:**

  1. Rename `/financier-summary` route to `/entity-summary` (update the GET handler)
  2. Call `debtService.getEntitySummary()` instead of `debtService.getFinancierSummary()`
  3. All other routes stay the same structure, just the service calls updated

  **IMPORTANT Decimal.js rules:**
  - Use `.plus()`, `.minus()`, `.times()`, `.dividedBy()` — NEVER `.add()`, `.sub()`, `.mul()`, `.div()`
  - Always: `new Decimal(dbValue.toString())` for Prisma Decimal fields
